<static> {Object[]} bui.util.calculateLabelPositioning(width, label, classes)
Calculate all the required information for the positioning of a label.
Parameters:
{Number} width
Available width for the positioning of the label
{String} label
The label for which the positioning should be calculated.
{String[]} classes Optional
An array of strings which represent css classes which should be applied to the DIV which is used for the calculation of word dimensions.
Returns:
{Object[]} An array of objects. Each object in this array represents one line. Each line object has a words property which itself is an array of all words of the label ans the respective dimensions of this word. Also, each line object has a maxHeight, totalWidth and horizontalIndention property.
<static> {Object[]} bui.util.calculateTextDimensions(text, classes)
Use this function to calculate the text dimensions for a line of text.
Parameters:
{String} text
Line of text to be analysed
{String[]} classes Optional
An array of strings which represent css classes which should be applied to the DIV which is used for the calculation of word dimensions.
Returns:
{Object[]} An object with word, width and height properties. For each word in the given text (text splitted at every whitespace character) the previously mentioned properties are returned.
<static> {Object} bui.util.calculateWordDimensions(text, classes, escape)
Calculate word dimensions for given text using HTML elements. Optionally classes can be added to calculate with a specific style / layout.
Parameters:
{String} text
The word for which you would like to know the dimensions.
{String[]} classes Optional
An array of strings which represent css classes which should be applied to the DIV which is used for the calculation of word dimensions.
{Boolean} escape Optional
Whether or not the word should be escaped. Defaults to true.
Returns:
{Object} An object with width and height properties.
<static> bui.util.createListenerTypeId()
All listener types must have a unique identifier. In the previous version strings were used as an identifier with the drawback of bad performance due to a fair amount of lookups. This function just generates an integer which should be much faster for lookups.
